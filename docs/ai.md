# Ren'Py + WorkBuddy 智能剧本集成与商业级动效工作流指南

> **目标**：建立一套自动化工作流，让 WorkBuddy AI 将用户新增的单文件剧本无缝追加到现有 Ren'Py 项目中，并自动注入商业级视觉动效，实现"丢文件 → AI处理 → 直接可运行"的闭环。

---

## 一、环境准备与工具安装

### 1.1 安装 WorkBuddy

根据网络环境选择以下任一方式安装 [WorkBuddy](https://cloud.tencent.com/developer/article/2658096) [^3^]：

| 安装方式 | 适用场景 | 操作步骤 |
|---------|---------|---------|
| **官方安装包** | 个人用户、快速上手 | 访问 WorkBuddy 官网下载对应系统安装包（Windows/macOS），按向导完成安装 |
| **SkillHub 在线安装** | 已安装客户端，扩展功能 | 打开 WorkBuddy → 左侧"技能" → SkillHub → 搜索所需技能 → 点击"+"安装 [^2^] |
| **私有化部署** | 企业团队、数据敏感 | 联系官方获取私有化部署方案，支持本地数据隔离与行为审计 [^3^] |

**必装技能清单**：
- `file-manager`：本地文件读写、目录扫描
- `code-explorer`：代码解析与批量编辑
- `agent-memory`：记忆项目结构与上下文
- `exec`：执行本地命令（调用 Ren'Py SDK）
- `git`：版本控制（可选但强烈建议）

### 1.2 安装 Ren'Py SDK

1. 访问 [Ren'Py 官网](https://www.renpy.org/) 下载最新稳定版 SDK
2. 解压到固定路径（建议 `C:\RenPy` 或 `~/RenPy`）
3. 将 SDK 路径添加到系统环境变量，方便 WorkBuddy 调用
4. 验证安装：在终端执行 `renpy --version`

## 二、WorkBuddy 自动化配置

### 2.1 创建"Ren'Py 剧本集成"专属技能

在 WorkBuddy 中创建自定义 Skill，实现一键触发：

**触发关键词**：`集成新剧本到RenPy`

**系统提示词（System Prompt）**：
```
你是一名 Ren'Py 视觉小说引擎的资深开发工程师，专精于剧本结构解析与商业级动效实现。

你的任务：
1. 扫描项目根目录下的 projects/ 文件夹，找到用户新增的单文件剧本
2. 解析剧本内容，识别角色、对话、场景、情绪标记
3. 将解析后的剧本以 Ren'Py 语法追加到 game/script/ 目录下的合适位置
4. 为剧本中的关键节点自动注入商业级 ATL 动效（转场、角色动画、UI动效、粒子特效）
5. 更新 game/script.rpy 的主入口，确保新剧本被正确加载
6. 生成处理日志到 log/integration_log.md

约束条件：
- 不得破坏已有剧本的任何现有逻辑
- 所有插入操作前必须创建备份（.bak）
- 动效代码必须遵循 Ren'Py ATL 语法规范
- 优先使用已定义的角色和图像资源，如遇到新角色需先在 characters.rpy 中注册
```

### 2.2 配置自动化任务（可选）

如需实现"文件一丢，自动处理"：

1. 打开 WorkBuddy → 左侧"自动化" → "+ 添加"
2. 配置如下：
   - **名称**：Ren'Py 剧本自动集成
   - **工作空间**：本地守护进程
   - **提示词**：`检测 projects/ 目录变化，如有新增 .txt/.rpy 文件，执行"集成新剧本到RenPy"技能`
   - **执行频率**：按间隔 / 每 5 分钟检测一次
3. 保存并启用

## 三、新增剧本格式规范（用户须知）

为了让 AI 正确解析，用户放置的剧本文件需遵循以下**轻量标记格式**：

```text
# 剧本标题：第三章 - 雨夜重逢
# 插入位置：chapter2.rpy 之后
# 优先级：normal

--- 场景：咖啡厅 ---
[背景：bg_cafe_night]
[音乐：bgm_melancholy]
[时间：夜晚]

林夏：（担忧）你……怎么会在这里？
[角色：林夏，表情：sad，位置：left]

顾辰：（苦笑）我倒是想问你，三年了，为什么还留着那枚戒指？
[角色：顾辰，表情：bitter_smile，位置：right]
[特效：rain_window]

林夏：（低头，沉默）……因为忘不掉。
[动作：林夏，动画：head_down]
[转场：fade，时长：1.5]

--- 场景：回忆闪回 ---
[背景：bg_university_sunset]
[滤镜：sepia]
[时间：回忆]

顾辰：（温柔，年轻声线）等我回来，我们就去海边。
[角色：顾辰，表情：gentle，位置：center]
[特效：sparkle]
[摄像机：zoom_in，目标：顾辰脸部，时长：2.0]

--- 场景：回到现实 ---
[背景：bg_cafe_night]
[音乐：bgm_melancholy，音量：80%]

林夏：（坚定）这次，我不会让你走了。
[角色：林夏，表情：determined，位置：center]
[动作：林夏，动画：step_forward]
[转场：dissolve，时长：1.0]
[结束]
```

**标记说明**：

| 标记 | 含义 | AI 处理动作 |
|-----|------|------------|
| `# 剧本标题` | 剧本标识 | 生成文件名，如 `chapter3.rpy` |
| `# 插入位置` | 目标锚点 | 确定在现有剧本链中的位置 |
| `--- 场景 ---` | 场景分隔 | 生成 `label` 节点 |
| `[背景：xxx]` | 背景图 | 转换为 `scene bg xxx` |
| `[音乐：xxx]` | BGM | 转换为 `play music xxx` |
| `[角色：name]` | 说话人 | 匹配 `characters.rpy` 中定义的角色对象 |
| `[表情：xxx]` | 立绘表情 | 匹配 `images/` 下对应表情图，注入 ATL 切换动效 |
| `[位置：xxx]` | 屏幕位置 | 转换为 `show char at xxx` |
| `[动作/动画：xxx]` | 角色动作 | 注入对应的 ATL 动画块 |
| `[转场：xxx]` | 场景过渡 | 转换为 `with xxx` 或自定义 ATL 转场 |
| `[特效：xxx]` | 环境特效 | 注入粒子或全屏特效 |
| `[摄像机：xxx]` | 镜头运动 | 注入 ATL 摄像机变换 |
| `[滤镜：xxx]` | 色彩滤镜 | 注入 `matrixcolor` 变换 |
| `[结束]` | 剧本终止 | 生成 `return` 或跳转标记 |

## 四、AI 集成执行流程

当用户触发 WorkBuddy 或自动化任务检测到新文件时，AI 按以下步骤执行：

### 4.1 步骤一：扫描与备份

```
1. 扫描 projects/ 目录，按修改时间排序获取最新文件
2. 读取文件内容，进行 UTF-8 编码检测
3. 对以下文件创建 .bak 备份：
   - game/script.rpy（主入口）
   - game/script/ 下与"插入位置"相关的文件
   - game/characters/characters.rpy（如需注册新角色）
```

### 4.2 步骤二：剧本解析与转换

AI 将用户的轻量标记转换为标准 Ren'Py 脚本：

**输入片段**：
```text
林夏：（担忧）你……怎么会在这里？
[角色：林夏，表情：sad，位置：left]
```

**AI 转换输出**：
```renpy
# 角色立绘以商业级淡入+微动效呈现
show linxia sad at left with dissolve

# 注入呼吸动效（商业级细节）
show linxia sad:
    subpixel True
    parallel:
        yoffset 0
        ease 2.0 yoffset -3
        ease 2.0 yoffset 0
        repeat
    parallel:
        alpha 1.0
        ease 0.3 alpha 0.95
        ease 0.3 alpha 1.0
        repeat

linxia "你……怎么会在这里？" with fadein_text
```

### 4.3 步骤三：主入口更新

AI 自动修改 `game/script.rpy`，确保新剧本被调用：

```renpy
label start:
    call chapter1
    call chapter2
    call chapter3  # ← AI 自动追加这一行
    return
```

### 4.4 步骤四：冲突检测

- 检查新剧本中的角色名是否在 `characters.rpy` 中定义
- 检查引用的背景图、音乐是否存在于 `images/`、`audio/` 目录
- 如发现缺失资源，在日志中标记为 `[WARN: 缺失资源]` 并生成占位符代码

## 五、质量检查与测试流程

### 5.1 AI 自动检查清单

集成完成后，WorkBuddy 自动执行以下验证：

- [ ] **语法检查**：调用 Ren'Py SDK 的 `renpy.sh lint` 命令
- [ ] **资源引用检查**：确认所有 `show`/`scene`/`play` 引用的资源存在
- [ ] **角色定义检查**：确认所有说话人已在 `characters.rpy` 中定义
- [ ] **标签冲突检查**：确认新 `label` 不与现有标签重复
- [ ] **备份确认**：确认 `.bak` 文件已生成

### 5.2 人工快速测试步骤

1. 打开 Ren'Py Launcher，选择项目
2. 点击 "Launch" 启动游戏
3. 使用 **Skip** 模式快速跳转到新章节
4. 检查以下要点：
   - 新剧本是否正确衔接上一章结尾
   - 角色立绘是否有呼吸/情绪动效
   - 转场是否平滑（非瞬间切换）
   - 粒子特效是否正常显示（不遮挡对话框）
   - 音乐/音效是否正确播放和切换
   - 摄像机运动是否流畅（无卡顿/抖动过度）

### 5.3 常见问题速查

| 现象 | 可能原因 | 解决方案 |
|-----|---------|---------|
| 新剧本未加载 | `script.rpy` 未更新 `call` 语句 | 检查 AI 是否成功修改主入口 |
| 角色不显示 | 角色名拼写与 `characters.rpy` 不一致 | 统一角色命名规范，区分大小写 |
| 动效卡顿 | 同时运行过多 `parallel` 动画 | AI 自动优化为 `subpixel True`，如仍卡顿可简化粒子数量 |
| 滤镜全黑 | `matrixcolor` 参数错误 | 检查 `filters.rpy` 中的矩阵乘法顺序 |
| 音乐未切换 | `play music` 前未 `stop music` | AI 已自动处理 fadeout/fadein |

## 六、进阶优化建议

### 6.1 性能优化

- **图集打包**：使用 Ren'Py 的 `build` 功能将 `images/` 打包为 `.rpa`，减少文件句柄
- **粒子优化**：限制同时显示的粒子数量（建议 ≤100），使用较小的纹理尺寸
- **ATL 优化**：避免在 `repeat` 动画中使用过于复杂的 `matrixcolor` 计算

### 6.2 多语言支持

如需后续支持多语言，AI 可在集成时同时生成：
- `game/tl/chinese/chapter3.rpy`
- `game/tl/english/chapter3.rpy`

只需在源剧本标记中增加 `[语言：zh,en]`，AI 自动拆分。

### 6.3 版本控制（Git）

强烈建议将项目纳入 Git 管理：

```bash
# 初始化仓库（如尚未初始化）
git init

# 创建 .gitignore
echo "*.rpyc
*.rpyb
log/
saves/
cache/" > .gitignore

# 每次 AI 集成后自动提交
git add .
git commit -m "WorkBuddy AI 集成: 新增 chapter3 - 雨夜重逢 + 商业动效"
```

WorkBuddy 的 `git` 技能可自动执行以上操作 [^5^]。

### 6.4 自定义动效扩展

如果现有动效库不满足需求，用户可在 `game/effects/custom.rpy` 中编写自定义 ATL，AI 在后续集成时会识别并引用这些自定义动效。
